## 3. 本次用例设计流程沉淀
### 3.1 设计节奏
- 先做需求澄清：不明确点先问清，不补脑。
- 再出单需求用例初稿：先保证功能主流程完整，再补异常、边界、中断。
- 然后做单个复查：检查异常场景是否缺失、边界值是否需要补充、优先级是否合理、标题/前置条件/步骤/预期是否可顺手优化。
- 再做中阶段优化：异常/边界补漏、优先级复查、文案收短、步骤收短。
- 最后做最终复查：检查逻辑重复、是否有可删可并但不强行合并的地方，先出结论清单，再改文件。

### 3.2 本次沉淀出的高频规则
- 录音大类需求默认拆成`通话录音 / 现场录音 / 音视频录音`三组。
- 如无系统差异，Android/iOS默认不拆；若系统交互不同，再单独分组。
- 单条用例最多`3`步，优先`2`步；尽量只验证`1`个核心断言。
- 不强行合并用例；若同一条同时覆盖两个独立断言，应拆开。
- 文案要能收短就收短，但不能收坏断言边界。
- 先统一结构，再做文案精简；避免边写边大范围改层级。

### 3.3 最终产物建议
- 正式用例文件：按需求拆分的`*_测试用例.md`。
- 汇总文件：`630测试用例.md`。
- 对照清单：`需求点_用例标题数量对照清单.md`。
- 简约版测试点：用于测试报告展示。
- 现网包验证重点：用于回归优先级排序。

## 4. 通用测试设计规则
### 4.1 结构与表达
- `pc:`优先写状态，不重复写“打开页面后”这类低信息量动作。
- 步骤写动作，预期写可观察结果，不混写结论和操作。
- 同一文件内术语统一，例如：`toast`、`弹窗`、`横幅`、`kill APP`、`最小化后切回前台`。

### 4.2 异常/边界常用模型
- 中断类：切后台、退出详情页、最小化、kill APP、锁屏/息屏。
- 网络类：断网、网络恢复、服务失败、云端下载失败。
- 交互类：快速连点、重复点击、置灰项点击、重复发起任务。
- 数据类：无内容、部分内容、历史数据、跨账号、跨重启、重装后。
- 批量类：数量上限、部分成功、全部无内容、单条失败不拖垮整体。

### 4.3 优先级判断经验
- `P0`：核心正向主流程，用户强感知且版本直接改动的成功路径。
- `P1`：常用功能、重要切换逻辑、关键兼容场景、重要错误/边界。
- `P2`：不常用场景、展示校验、恢复校验、扩展路径。
- `P3`：反向、失败、极端边界、容错、互斥校验。

- 有转写 / 无转写 / 转写失败 / 转写无内容。
- 有摘要 / 无摘要。

## 6. 功能知识沉淀
### 6.1 录制过程中可暂停录制
#### 核心规则
- 三类录音都要覆盖暂停、继续、结束保存。
- 暂停后要区分“仅录音”和“开启实时转写”两种恢复状态。
- 暂停期间切换转写语言、翻译语言，恢复录音后应立即按新语言生效。
- 暂停后最小化返回，录音保持暂停态；暂停后`kill APP`，录音应结束并保存。

#### 重点验证点
- 转写语言、翻译语言、文字直播设置弹窗交互。
- 区分说话人按钮隐藏、通话录音中嘴替入口隐藏。
- 网络异常 / 双连丢包横幅提示与恢复。
- 浅色 / 深色 / 英文模式样式。

#### 易漏点
- 锁屏/息屏恢复。
- 快速连点暂停/继续后的状态一致性。
- 暂停状态下尝试并发新录音。
- 暂停后耳机断连自动结束并保存。

### 6.2 导出功能优化
#### 核心规则
- 详情页导出弹窗固定展示5个按钮：`导出音频+转写 / 导出音频 / 导出转写 / 导出摘要 / 导出思维导图`。
- 按内容存在性决定可用/置灰，不因无内容隐藏按钮。
- 摘要与脑图Tab支持直接导出。
- 批量导出遵循内容过滤、数量上限、单条失败不中断整体。

#### 文件命名规则
- 单文件导出命名：
  - `记录名称_Audio_YYYY-MM-DD HH_MM_SS`
  - `记录名称_Transcript_YYYY-MM-DD HH_MM_SS`
  - `记录名称_Summary_YYYY-MM-DD HH_MM_SS`
  - `记录名称_Mind map_YYYY-MM-DD HH_MM_SS`
- zip命名：
  - 单条音频+转写zip：`viaim_YYYY-MM-DD HH_MM_SS.zip`
  - 批量导出zip：`viaim_YYYY-MM-DD HH_MM_SS.zip`
- 转写时间戳开关只影响导出内容，不影响文件名。

#### 离线导出规则
- 支持离线：`txt / mp3 / jpeg / markdown / xmind`
- 不支持离线：`docx / pdf`
- 断网选择`docx/pdf`需提示`请联网后重试`

#### 批量导出规则
- 选择`51`条及以上时，点击导出才提示上限。
- 批量导出摘要按内容存在性过滤记录。
- 批量导出音频+转写仅导出同时具备音频和转写的记录。
- 批量导出思维导图时`jpeg`置灰并提示`不支持批量导出`。
- 无批量下载按钮；重装后不再弹“是否下载全部记录”提示。

#### 易漏点
- 云端未下载文件先下载再导出。
- 个别文件失败不应中断整批。
- 停止导出 / kill APP后不留残损文件。
- 进度弹窗期间禁止切换其他模块。

### 6.3 录音记录中标记增加时间戳
#### 核心规则
- 点击标记按点击时刻前后`10s`生成时间戳区域。
- 开始不足10秒和结束不足10秒时按边界截断。
- 支持多次独立标记，详情页音频条与转写高亮对应展示。

#### 交互规则
- 已下载后才能展开音频条并进行标记点点击/拖拽播放。
- 标记区域转写默认高亮为黄色，播放中的对应高亮为橙色。
- 点击标记区域或拖拽到标记区域，应能定位播放。

#### 兼容规则
- APP端标记后，Web端与桌面端展示应一致。
- 仅时间戳标记时重新转写不弹清除标记弹窗，且高亮保留。
- 含文本类型标记时重新转写需要弹出清除标记提醒。
- 重新转写失败不应丢失原有时间戳标记。

### 6.4 非实时转写可自动识别语种
#### 核心规则
- 三类录音的“音频转文字”入口都新增并默认选中`自动识别语种`。
- 自动识别需覆盖多语种样本和中英混合样本。
- 识别不到有效语言时，要区分“默认输出中文”与“提示转写无内容”的场景。

#### 默认语种逻辑
- 优先级：`当前记录已存语种（不含自动） > 本地持久化非实时转写语种（含自动） > 自动`
- 非实时转写语种与实时转写历史独立持久化。
- 安装后首次使用非实时转写时：
  - 若实时转写开启，默认带入当前实时转写语种。
  - 若实时转写关闭，默认选中`自动`。
- 同一录音若自动识别结果已落成具体语种，再次转写默认带入该具体语种。

#### 中断规则
- 切后台、退出详情页、kill APP、服务失败、超长失败、切换账号，任务都应终止并恢复安全状态。

### 6.5 翻译在详情页显示
#### 核心规则
- 仅“已有转写内容”的详情页可正常发起翻译；无转写、转写为空、转写失败时入口展示但置灰。
- 选择翻译语言后应立即收起弹窗并开始翻译。
- 翻译结果逐段展示在转写下方，语言展示正确。

#### 翻译中规则
- 顶部固定展示`正在翻译...`。
- 翻译中禁止音频转文字、区分说话人等记录操作，尝试操作需提示`正在翻译中，请稍后再试`。
- 重新翻译进行中但尚未成功时，页面继续展示旧翻译。
- 返回上一页或`kill APP`都应终止翻译。

#### 离线规则
- 断网仅支持中文、英语离线翻译。
- 断网选择其他语种需提示`网络走丢了`。

### 6.6 全局搜索点进详情页定位关键词
#### 核心规则
- 搜索范围包含：标题、转写、翻译。
- 同一条记录多处命中时，只展示一张卡片。
- 命中来源优先级：`标题 > 转写 > 翻译`。

#### 详情页定位规则
- 标题命中：进入详情页，标题不高亮。
- 转写/翻译命中：进入详情页后自动切到转写Tab并执行页内查找。
- 内容命中支持上下切换、首尾循环、保持当前命中可视。
- 多来源同时命中时，点击卡片忽略标题，直接定位首个内容命中。

#### 易漏点
- 搜索词切换后旧结果清空。
- 清空关键词后旧结果立即清空。
- 返回搜索结果页时保留原关键词和结果。
- 详情页已有旧查找状态时，应被新搜索状态替换。

### 6.7 语音控制耳机文案修改
#### 核心规则
- `air2`与`pro3`都要覆盖。
- 固件未升级时展示旧唤醒词：`开始录音 / 停止录音`。
- 固件升级后展示新唤醒词：`你好小万，开始录音 / 你好小万，停止录音`。
- 新旧文案必须互斥，不能混触发。

#### 易漏点
- 停留在语音控制页切换不同固件版本耳机时，文案和能力需实时刷新。

### 6.8 新增隐藏的用户高级设置
#### Android规则
- 缩减版入口：长按`viaim`图标`10s`或快速连续点击`关于我们`标题`10`次。
- 完全版入口：快速连续点击公司名称文案`10`次。
- 缩减版包含：更换面翻通道、导出日志、手动升级固件、恢复记录。
- 恢复记录：未输入文件名时展示`请输入文件名`且按钮置灰；不存在文件名提示`文件名错误请检查重试`。

#### iOS规则
- 缩减版入口：长按`viaim`图标`10s`。
- 完全版入口：快速连续点击公司名称文案`10`次。
- 缩减版包含：导出日志、手动升级固件、导出备份音频。

#### 通用规则
- 达到阈值后仅进入一次，不重复弹出。
- 中断后计时/计数清零，不累计。
- 子功能本期只做正向可用性回归，失败恢复做轻量覆盖。

### 6.9 邀请内测用户
#### 核心规则
- 非白名单：不展示邀请弹窗，也不展示`首席体验馆`入口。
- 白名单：升级后自动弹窗仅一次，按账号维度控制。
- 标题、内容、按钮文案均支持服务端下发；断网时走客户端兜底。

#### 按钮行为
- `立即申请`：跳转企业微信名片；失败不应误更新状态，后续仍可申请。
- `稍后考虑`：关闭弹窗，保留`首席体验馆`入口。
- 点击弹窗外空白：与`稍后考虑`一致。
- `我没兴趣，不再提示`：后续不再自动弹窗，也不展示`首席体验馆`入口，恢复`你好小万`入口。

#### 状态规则
- 关闭弹窗后跨重启不再自动弹出。
- 点击`首席体验馆`入口可再次唤起弹窗，入口本身不消失。
- 账号切换后状态互相隔离。

### 6.10 合并录音
#### 范围规则
- 本期仅支持手动合并，不做自动合并。

#### 前置校验优先级
- 不支持条件提示优先级：
  - 翻译模式录音不支持合并
  - 超过10条
  - 总时长超过5小时
  - 含未下载音频且断网
  - 含处理中记录
- 多个限制同时存在时，只弹一个最高优先级toast。
- 当天成功合并`100`次后，第`101`次受限。

#### 确认页与标题规则
- 默认按时间从早到晚排序，支持拖拽手动调整顺序。
- 默认标题按第一条记录生成；若原标题已以“合并”或“合并+数字”结尾，按“合并2”“合并3”递增。
- 标题可编辑，清空后仍允许合并，合并后回退默认标题。
- `合并后保留原音频`开关默认关闭且不记忆。

#### 智能标题规则
- 智能标题开关默认关闭，但记忆上次状态。
- 点亮后标题输入框置灰，再次点击或点标题输入框可恢复编辑。
- 断网时点击点亮智能标题需toast，但仍允许点亮。
- 断网且点亮状态下点击合并，会再次toast，并按手动/默认标题继续合并。
- 联网时若智能标题生成失败，也回退到手动/默认标题。

#### 合并结果规则
- 音频和转写按用户设定顺序拼接，时间戳重新计算。
- AI内容基于完整转写重新生成。
- 类型、语种、地点、文件夹归属按第一条记录继承。
- 保留原始音频基本信息，便于溯源。

#### 特殊场景
- 含未下载音频且联网时，可先下载再合并。
- 含未转写录音或转写无内容录音，合并后按原状处理或直接拼接。
- 说话人设置不一致时保持原状。
- 部分音频无opus、部分有opus时，后续重转写/区分说话人统一使用mp3。
- 未联网且多选已上云记录、且未勾选保留原音频时，合并成功后需提示原记录删除失败。

## 7. 现网包回归重点
### 7.1 核心主流程优先
- 录音启停、暂停、恢复、结束保存。
- 自动识别语种默认值与中英识别结果。
- 详情页翻译结果展示。
- 全局搜索进入详情页定位。
- 导出与合并两个强感知功能。

### 7.2 强感知兼容点
- 固件前后语音控制文案切换。
- 老录音查看、历史标记兼容。
- 白名单弹窗与隐藏入口是否可达。

### 7.3 现网包易翻车场景
- 长任务中断后状态恢复。
- 云端文件与本地状态混用。
- 新功能对旧数据的兼容。
- 文案更新后能力与展示不一致。

## 8. 后续版本可直接复用的检查清单
- 是否需要按`通话 / 现场 / 音视频`三类录音拆分。
- 是否存在`入口展示 -> 可用/置灰 -> 执行 -> 结果 -> 中断 -> 恢复`完整链路。
- 是否覆盖断网、服务失败、切后台、kill APP、重进页面、重试。
- 是否覆盖本地/云端、已下载/未下载、部分成功/全部失败。
- 是否存在历史数据、新旧文案、新旧固件、新旧标记兼容问题。
- 是否需要补一份简约版测试点和现网包验证重点，便于报告和回归。

# 通话翻译功能规格（国内版 viaim App）

> **国内版专属功能**，海外版无对应功能。
>
> 版本：V1.0.0 | 最后更新：2026-04
>
> 官方页内称「**通话语音翻译**」（Call Voice Translation），简称「通话翻译」。适用：用户与对方进行系统通话/网络通话时，App 实时双向语音翻译。

---

## 一、功能定位

通话翻译是国内版 viaim App 专属的通话即时互译功能：
- **通话场景**：用户正在进行通话（系统通话 / 网络通话，类型⚠️ 待确认）
- **双向翻译**：我方说的话翻译为对方语言，对方说的话翻译为我方语言
- **嘴替机制**：App 作为"翻译助理"，通过 TTS 将翻译结果播报给对方；我方通过耳机听到对方原声 + 翻译
- **左右耳分工**（基于 Pro3/Pro2/Air2 等双耳耳机）：
  - **左耳机**：听到对方的**原声**
  - **右耳机**：作为**翻译助理**，播放翻译后的语音

> 因为依赖左右耳分工，**必须双耳同时佩戴**才能使用。

与同传听译 / 面对面翻译的对比见 `README.md` §一。

## 二、入口

入口位于 **首页底部录音按钮点击后弹出的录音模式选择弹窗 → 翻译 Tab → 通话翻译**（见 `../../home.md` §3.5.1）。

> ⚠️ 是否另有独立入口（如通话录音页内子入口、耳机手势直接触发）待确认。

## 三、前置校验链

点击入口后按以下顺序校验，任一未通过立即拦截：

| # | 校验项 | 未通过时的处置 |
|---|-------|--------------|
| 1 | **当前是否有通话** | Toast 提示并**拦截**（无通话时不允许进入通话翻译）|
| 2 | **是否双耳佩戴** | 弹窗「**请佩戴左+右耳机**」+ 文案「需要同时佩戴左右耳机才能使用通话翻译。」+「知道了」按钮，拦截进入 |
| 3 | 权限/耳机型号等其他前置 | ⚠️ 待确认（型号是否限制 / 麦克风权限 / 通话权限）|

> 图示已明确：在 `L:盒内 R:盒内`（即未佩戴）状态下点击通话翻译，会弹出上述拦截弹窗。

### 3.1 Toast 与弹窗文案

| 场景 | 文案 |
|------|------|
| 无通话 | ⚠️ 待确认具体 Toast 文案（如「请在通话中使用」）|

## 四、进入流程

> 前置校验通过后，**每次**进入通话翻译都遵循如下固定流程（不是仅首次）：

```
点击通话翻译入口
   │
   ├─ 校验有通话 + 双耳佩戴（§三）
   │
   ▼
① 语言选择弹窗（我方/对方语言选择）
   │   点击「确定」
   ▼
② 功能介绍页（「关于通话语音翻译」）
   │   点击「知道了」
   ▼
③ 通话翻译主页（开始使用）
```

### 4.1 每次都展示 vs 仅首次展示

- **语言选择**：每次进入都弹出（确认当前通话使用的语言对）
- **功能介绍**：**每次进入都展示**（不是首次限定）——这是通话翻译的特有设计，因每次通话场景/对方可能不同，需要重新确认耳机分工规则
- 是否有"本次不再展示"勾选项 ⚠️ 待确认

## 八、语音嘴替功能

**语音嘴替**是通话翻译的核心特性之一：App 作为翻译助理，把文字/识别结果转为对方语言的语音**替用户说**给对方。

| 模式 | 说明 |
|------|------|
| **自动嘴替**（默认）| 识别我方说话 → 自动翻译 → 自动播放给对方 |
| **手动输入嘴替**（顶部键盘图标）| 点击键盘图标 → 输入文字 → 翻译后播报给对方（不说话也能替用户发声）|

⚠️ 待确认细节：
- 键盘输入模式的 UI（弹键盘 / 全屏输入页 / 底部面板）
- 输入字数上限
- 是否支持预设常用短语（如截图中的"Hello, my spoken language is not very good..."可能就是预设或历史）
- 自动嘴替是否可关闭（应有对应开关 / 快捷切换）
- 与常规说话翻译的切换时机

## 九、设置弹窗

通过顶部齿轮图标唤起，底部弹出式设置面板。

### 9.1 设置项

| 项 | 类型 | 默认值 | 说明 |
|----|------|-------|------|
| **使用指南** | 跳转 `>` | — | 进入使用指南页（⚠️ 待确认是 H5 / 内部页）|
| **收听对方翻译** | 开关 | ✅ 开启 | 关闭后，您只会听到对方的**原声**；对方仍会听到您的翻译播报 |
| **朗读速度** | 5 档单选：更慢 / 慢 / 正常 / 快 / 更快 | ⚠️ 待确认（截图选中"慢"）| 翻译播报的 TTS 速度 |
| **字号大小** | 5 档单选：小 / 标准 / 大 / 特大 / 巨大 | 标准 | 翻译气泡文字字号 |

### 9.2 「收听对方翻译」开关的影响矩阵

| 状态 | 我方听到 | 对方听到 |
|------|---------|---------|
| 开启（默认）| 对方原声（左耳）+ 对方翻译（右耳）| 我方翻译（一直） |
| 关闭 | **仅**对方原声（左耳）| 我方翻译（不受影响）|

> 单向关闭，不影响对方端的翻译播报——这是"我只想练听力"场景的设计。

### 9.3 字号与朗读速度的生效范围

- 字号：仅影响当前通话翻译页的文字气泡
- 朗读速度：对方侧听到的 TTS 速度（⚠️ 是否也影响我方听到的对方翻译待确认）
- 是否跨翻译功能共享（例如设同一速度用于同传/面翻）⚠️ 待确认

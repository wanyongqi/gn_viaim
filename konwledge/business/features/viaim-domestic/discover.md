# 发现页功能规格（国内版 viaim App）

> 版本：V2.0.0 | 最后更新：2026-04
> 对应底部导航第三个 Tab「发现」
> **页面内容大部分由服务端/后台配置**；客户端主要负责容器与跳转能力。

---

## 一、页面概述

「发现」是国内版 viaim App 专属的 Tab（海外版无此 Tab），承载运营内容与产品相关的外链入口：
- 顶部 **2×2 快捷入口矩阵**（4 个固定入口，跳转目标可配）
- 中部与底部 **运营/活动横幅**（由后台配置）
- 右上角 **放松音乐入口**（需连接耳机才能进入）

> 大多数模块的**跳转内容、顺序、显隐**由服务端配置，客户端不应硬编码具体运营位内容；测试关注点以容器行为为主。

---

## 二、顶部导航栏

| 位置 | 元素 | 交互 |
|------|------|------|
| 居中 | 标题「发现」| 静态文字 |
| 右上 | 放松音乐入口（圆形图标）| 点击进入「放松空间」放松音乐页面（见 §五）|

## 三、2×2 快捷入口矩阵

顶部固定展示 4 个快捷入口，排列为 2 行 × 2 列：

| 位置 | 入口 | 图标 | 跳转目标 |
|------|------|------|---------|
| 左上 | **用户社区** | 人形图标 | 跳转到**微信页面**（微信小程序/公众号/社群，后台配置具体 URL）|
| 右上 | **功能更新** | 上箭头图标 | 跳转到 **H5 页面**（后台配置 URL，展示版本更新说明）|
| 左下 | **产品说明书** | 书本图标 | 跳转到 **H5 页面**（后台配置 URL）|
| 右下 | **产品选购** | 购物车图标 | 跳转到**微信页面**（微信小程序电商，后台配置）|

### 3.1 跳转类型汇总

| 跳转类型 | 入口 | 行为 |
|---------|------|------|
| **微信跳转** | 用户社区 / 产品选购 | 拉起微信，进入对应小程序/公众号/会话 |
| **H5 跳转** | 功能更新 / 产品说明书 | App 内置 WebView 打开 H5 页面 |

### 3.2 跳转异常

| 场景 | 预期行为 |
|------|---------|
| 未安装微信（跳微信类入口）| ⚠️ 待确认：弹出提示 / 静默失败 / 引导安装 |
| 无网络（跳 H5）| ⚠️ 待确认：WebView 显示错误页 / Toast 提示 |
| 后台未配置 URL | ⚠️ 待确认：入口是否隐藏 / 点击无效 |
| 服务端返回异常 URL | ⚠️ 待确认：客户端是否做 URL 合法性校验 |

## 四、运营/活动横幅

### 4.1 内容来源

- 全部由**后台配置**：图片 / 文案 / 跳转 URL / 展示时段 / 展示顺序 / 是否可关闭
- 客户端按后台下发内容渲染，不对具体活动做测试固化

### 4.2 常见展示形式（基于截图示例）

| 形式 | 说明 |
|------|------|
| 大图横幅 | 如「OXFORD TEST OF ENGLISH Advanced」这种全宽运营图，点击跳转后台配置 URL |
| 底部浮层横幅 | 如「耳机丢失无忧服务」，带关闭按钮（×），可手动关闭；关闭后 ⚠️ 待确认是否永久隐藏 / 下次启动重现 |
| 其他形式 | 卡片列表 / 轮播 / 视频等，按后台配置决定 |

### 4.3 通用规则

- **展示顺序**：由后台排序字段控制
- **展示时段**：过期后自动下架
- **跳转类型**：支持微信 / H5 / App 内跳转（见 §3.1）
- **埋点**：⚠️ 待确认运营位曝光/点击埋点规则

## 五、放松音乐（放松空间）

### 5.1 入口与前置条件

| 项 | 说明 |
|----|------|
| 入口 | 发现 Tab 右上角圆形图标 |
| **前置条件** | **必须连接耳机**才能进入 |
| 未连接耳机点击 | ⚠️ 待确认：提示"请先连接耳机" / 引导至首页连接 / 按钮置灰 |

### 5.2 页面结构（「放松空间」）

| 区域 | 元素 | 说明 |
|------|------|------|
| 左上 | `<` 返回按钮 | 返回发现 Tab |
| 居中顶部 | 标题「放松空间」+ 进度指示点（●●●●）| 指示点数量 = 当前可切换的主题/曲目数量 |
| 中央 | 主题音乐名称（如「柔软的主题音乐」）| 当前选中的主题 |
| 中央下方 | 播放按钮（▶）| 点击开始播放 / 暂停 |
| 底部 | 爱心图标 | 收藏当前主题音乐（⚠️ 收藏存储位置/同步规则待确认）|
| 底部 | 定时图标（含禁用标识）| 定时关闭功能（⚠️ 完整规则待确认）|
| 切换主题 | ⚠️ 待确认：左右滑动 / 点击指示点 / 其他 |

### 5.3 音频来源

- **所有音频文件均由管理后台配置**
- 客户端不内置固定曲库，按后台下发的列表动态渲染
- 音频格式 / 播放器行为 ⚠️ 待确认

### 5.4 播放行为

| 场景 | 行为 |
|------|------|
| 耳机断开 | ⚠️ 待确认：自动暂停 / 切到手机扬声器 / 弹窗提示 |
| 来电/其他 App 占用音频焦点 | ⚠️ 待确认：自动暂停恢复规则 |
| 切出放松空间页（返回 / 切 Tab）| ⚠️ 待确认：后台继续播放 / 自动暂停 |
| 定时关闭倒计时结束 | ⚠️ 待确认：停止播放 / 退出页面 |

## 六、与其他模块的关系

| 关系 | 说明 |
|------|------|
| 与底部 4 Tab | 独立切换，发现 Tab 不影响其他 Tab 状态 |
| 与耳机连接状态 | **强相关**：右上放松音乐入口需耳机连接才能用 |
| 与云空间开关 | **无关**：云空间开/关不影响发现 Tab 内容 |
| 与登录状态 | ⚠️ 待确认：未登录是否可见发现 Tab / 运营位是否展示 / 是否需登录才能跳转微信或 H5 |
| 与网络状态 | 强相关：所有运营内容依赖服务端下发，无网时 ⚠️ 待确认展示形式（缓存 / 空态 / 错误提示）|

---

## 七、异常与边界场景

- **后台配置为空**：发现页是否展示骨架页 / 提示页 / 静默 ⚠️ 待确认
- **后台配置耗时加载**：页面首次进入的 loading / 骨架屏样式 ⚠️ 待确认
- **跳转 URL 无效 / 404**：客户端容错处理 ⚠️ 待确认
- **微信 SDK 初始化失败**：跳微信类入口的降级 ⚠️ 待确认
- **放松音乐主题音频下载失败**：提示 / 重试机制 ⚠️ 待确认

## 八、测试关注点

### 8.1 容器层（客户端行为）

- 发现 Tab 切换进入，顶部 2×2 入口排列正确、图标文案正确
- 4 个快捷入口的跳转类型：
  - 用户社区 → 微信（验证能否正常拉起微信）
  - 产品选购 → 微信
  - 功能更新 → App 内 WebView
  - 产品说明书 → App 内 WebView
- 未安装微信时跳微信类入口的降级表现
- 无网/弱网下 H5 页面加载表现
- 运营位（大图/底部浮层）：后台配置什么就展示什么，关闭按钮可关闭
- 底部浮层关闭后是否永久隐藏 / 下次启动重现
- 底部 4 Tab 独立切换，切出再回来页面状态（运营位是否重新请求、音乐是否保持）

### 8.2 放松音乐

- **前置条件**：未连接耳机时入口不可用（核心边界）
- 已连接耳机：
  - 进入「放松空间」
  - 主题列表从后台下发，进度指示点数量匹配
  - 切换主题方式（滑动/点击指示点）
  - 播放/暂停切换
  - 收藏功能（爱心点亮/取消）
  - 定时关闭（⚠️ 完整规则落地后补）
- 播放中断场景：来电、切 Tab、切出 App、耳机断开
- 定时关闭倒计时结束的停止行为

### 8.3 跨模块

- 云空间开关开/关，发现页展示**完全一致**（核心回归点）
- 切换语言（中文 ↔ English）后 4 个快捷入口文案同步切换；跳转 URL 是否按语言分别配置 ⚠️ 待确认
- 登录/未登录状态下发现 Tab 的差异

13. 发现 Tab 在无网时的展示（缓存 / 空态 / 错误）

# 首页功能规格（国内版 viaim App）

> 版本：V2.0.0 | 最后更新：2026-04
> 对应底部导航第一个 Tab「首页」
> **首页展示仅与耳机连接状态有关，与云空间开关无关。**

---

## 一、页面概述

首页是国内版 viaim App 的主入口，核心职责是**展示当前耳机状态并提供耳机相关的快捷操作**。未连接耳机时引导用户完成连接；连接后呈现实时电量、佩戴状态、主耳信息和常用耳机设置快捷入口，并提供一键录音能力。

> 耳机连接流程详见 `bluetooth-connect.md`；各型号耳机支持的功能矩阵见 `bluetooth-headset.md`。

---

## 二、顶部导航栏（两种状态通用）

| 位置 | 元素 | 交互 |
|------|------|------|
| 左上 | 消息中心入口（信封图标）| 有未读时显示红色数字角标（如 `2`）；点击进入消息中心 |
| 居中 | 标题「首页」| 测试环境展示「首页（测试环境!!!）」以区分正式环境 |
| 右上 | 扫码入口（扫描图标）| 扫码登录到 **viaim 桌面端 / viaim 云空间 Web**（与 `login-register.md` 中的 App 扫码登录能力联动）|

## 三、耳机连接状态展示

### 3.1 状态总览

| 状态 | 首页主体展示 | 主 CTA |
|------|------------|-------|
| **未连接** | viaim logo + 耳机收纳盒示意图 | 底部「开始连接」按钮 |
| **连接中** | ⚠️ 待确认（过渡态 / loading 表现）| ⚠️ 待确认 |
| **已连接** | 耳机图 + 电量 + 主耳 + 佩戴状态 + 4 快捷入口 + 录音按钮 | 大红色录音按钮（●）|

### 3.2 未连接状态

- 中央区域：viaim logo + 耳机收纳盒图
- 底部主按钮：「开始连接」→ 跳转到耳机连接流程（见 `bluetooth-connect.md`）
- 不展示电量/佩戴/快捷入口/录音按钮

### 3.3 已连接状态

| 区域 | 元素 | 说明 |
|------|------|------|
| 顶部条件栏 | 「耳机固件可以升级啦！」| **仅在有新固件时显示**；点击进入固件升级流程（见 `bluetooth-headset.md` §固件升级） |
| 中央耳机图 | 左右耳示意图 + 主耳标识（红点）| 红点显示在当前作为发声耳/主耳的一侧 |
| 电量 | 绿色电池图标 + 百分比 + 耳机盒电量 | 展示规则按**耳机所在状态**决定（见 §3.3.1）|
| 主耳信息 | 「主耳：左/右」 | 按当前主耳动态显示 |
| 佩戴状态 | `L:盒外 R:入耳` 格式 | 左右耳分别标注：盒外 / 入耳 / 盒内（⚠️ 完整枚举待确认）|

#### 3.3.1 电量展示规则

| 耳机所在状态 | 电量展示内容 |
|------------|-------------|
| **双耳全部入盒 + 盒子打开** | 单耳电量（取左右耳中**较高者**）+ 耳机盒电量 |
| **至少一只耳机取出（盒外/入耳）** | 左耳电量 + 右耳电量 + 耳机盒电量（共 3 个数值分别展示）|
| 双耳全部入盒 + 盒子关闭 | ⚠️ 待确认（通常此时 App 无法读取耳机状态）|

### 3.4 快捷入口（已连接状态，4 个横向排列）

| 入口 | 图标 | 点击效果 |
|------|------|---------|
| **连接管理** | 蓝牙图标（可带未读红点角标）| 进入蓝牙连接管理页，等同于海外版「蓝牙连接管理」，规则见 `bluetooth-headset.md` §蓝牙连接管理 |
| **声音风格**（当前名称显示，如「大师古典」）| 音效图标 | 进入声音风格设置，与海外版一致，规则见 `bluetooth-headset.md` §3.4 声音风格 |
| **噪声控制 / 操作控制**（按型号动态切换，见下）| 人形图标 | 进入对应设置页，与海外版一致 |
| **更多** | 三点/省略号图标 | 进入完整耳机设置页（其他设置项） |

> **设计原则**：这 4 个入口的**点击效果与海外版耳机「连接管理/耳机设置」里的功能完全一致**，国内版仅在首页层重新组织了入口位置。各功能的具体规则不在本文件重复描述，统一引用 `bluetooth-headset.md`。

#### 3.4.1 第 3 个入口的按型号降级

| 耳机型号 | 是否支持噪声控制 | 首页该位置展示 |
|---------|---------------|--------------|
| Pro3 / Pro2 / Nano+ | ✅ | **噪声控制**（文案随当前档位实时更新：关闭 / 降噪 / 通透）|
| Air2 / VIAIM Air | ❌ | **操作控制**（不支持降噪的型号降级展示此入口）|

> 本规则依据 `bluetooth-headset.md` §3.3 各型号噪声控制支持情况；"操作控制"具体包含哪些设置项 ⚠️ 待确认。

### 3.5 录音按钮

| 项 | 说明 |
|----|------|
| 位置 | 已连接状态下的底部中央 |
| 样式 | 大红色圆形按钮（●）|
| 点击效果 | 弹出「**录音模式选择弹窗**」（非直接开始录音）|

#### 3.5.1 录音模式选择弹窗

弹窗内含两个 Tab：

| Tab | 内容 | 对应功能文档 |
|-----|------|------------|
| **录音** | 3 种录音模式（通话 / 现场 / 音视频）| `recording/README.md` |
| **翻译** | 3 种翻译模式（同传听译 / 面对面翻译 / 通话翻译）| `recording/translate.md` + `translation/README.md` |

- Tab 可手动切换，切换后选择的模式决定后续进入哪个功能
- 「翻译」Tab 是国内版**专属布局**：把同传听译与 2 个国内独有翻译功能聚合到一个入口
- 具体弹窗 UI（默认 Tab / 模式顺序 / 带图标说明）⚠️ 待补充
- 弹窗关闭方式遵循 `domain/common-interaction.md` 的通用规则

## 四、运营位（首席体验官等）

| 项 | 说明 |
|----|------|
| 位置 | 首页右下角悬浮 |
| 本质 | **运营位**，由后台配置（图标 / 文案 / 跳转 / 是否展示）|
| 展示形态示例 | 图标 + 文案（如「首席体验官」）+ 关闭按钮（×）|
| 关闭行为 | 由后台配置决定（单次关闭有效 / 永久关闭 / 下次启动重置 ⚠️ 按配置确定）|
| 点击跳转 | 由后台配置的目标 URL/页面决定 |

> **测试策略**：运营位的**具体内容**不作为测试用例核心（随后台变动），但应覆盖「后台开启/关闭展示」「关闭按钮点击后消失」「跳转 URL 可达」等容器层行为。

## 六、与其他模块的关系

| 关系 | 说明 |
|------|------|
| 与云空间开关 | **无关**——无论云空间开启/关闭，首页展示完全一致 |
| 与耳机连接状态 | **强相关**——未连接/已连接是首页展示的**唯一主变量** |
| 与登录状态 | ⚠️ 待确认：未登录时首页是否可见 / 是否有区别展示 |
| 与消息中心 | 左上信封图标 + 未读角标，规则见 `message-center.md` |
| 与固件升级 | 有新固件时顶部横幅提示，点击进入升级流程 |
| 与录音能力 | 已连接状态下，底部红色录音按钮是**首页唯一**的录音/翻译启动入口；弹窗含「录音」「翻译」两个 Tab |
| 与桌面端/Web | 右上扫码入口用于扫码登录 viaim 桌面端 / viaim 云空间 Web |

## 七、异常与边界场景

- **耳机连接后又断开**：首页从"已连接态"回到"未连接态"，中央显示 logo + 收纳盒图，主 CTA 切回「开始连接」
- **耳机电量低（< 30%？具体阈值 ⚠️ 待确认）**：首页电量数字是否变色/加提示 ⚠️ 待确认
- **固件升级完成后**：顶部横幅是否立即消失 ⚠️ 待确认（预期下次检测无新版本后消失）
- **同时连接多只耳机（双设备）**：首页展示哪一只的电量/佩戴状态 ⚠️ 待确认
- **测试环境标识**：仅测试包显示「（测试环境!!!）」；生产包不展示

## 八、测试关注点

- **连接状态切换**：未连接 → 连接中 → 已连接 → 断开 的完整 UI 转换
- **电量展示按所在状态**：
  - 双耳全部入盒+盒子打开：单耳电量（较高者）+ 盒电量（验证取较高而非较低）
  - 有耳机取出：左耳电量 + 右耳电量 + 盒电量（3 项）
  - 盒子关闭后的电量刷新行为 ⚠️ 待确认
- **主耳切换**：单耳佩戴 vs 双耳佩戴时主耳标识是否动态更新
- **佩戴状态组合**：双入耳 / 单入耳 / 双盒外 / 一入耳一盒外 的 UI 正确性
- **4 快捷入口**：每个入口点击跳转的目标页面正确性（与 `bluetooth-headset.md` 对应功能一致）
- **第 3 入口按型号降级**：
  - Pro3 / Pro2 / Nano+ 连接时展示「噪声控制」+ 当前档位（关闭/降噪/通透）
  - Air2 / VIAIM Air 连接时展示「操作控制」
- **声音风格入口文案**：入口文案随当前选中风格实时更新
- **录音按钮与录音模式弹窗**：
  - 未连接耳机：按钮不展示
  - 已连接：点击弹出含「录音」「翻译」两 Tab 的选择弹窗
  - Tab 手动切换功能
  - 3 大翻译 Tab 内是否正确列出：同传听译 / 面对面翻译 / 通话翻译
- **扫码入口**：扫码登录到桌面端 / 云空间 Web 成功率，扫码页权限申请，失败提示
- **固件升级横幅**：有/无新固件时的显示与消失，升级失败后是否保留入口
- **消息中心红点**：有未读时红点显示；阅读后红点消失
- **运营位**（首席体验官等）：后台开启/关闭的展示控制，关闭按钮点击后消失，跳转 URL 可达
- **底部 Tab 独立切换**：切到其他 Tab 后切回首页，耳机连接状态 UI 应实时反映当前真实状态
- **云空间开关无关性**：关闭云空间后首页展示无任何变化（核心回归点）

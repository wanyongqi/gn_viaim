# 录音记录列表功能规格（国内版 viaim App）

> 版本：V1.2.0 | 最后更新：2026-04

---

## 一、展示范围与分组视图

仅展示**音频类型**记录（耳机录音、闪录下载）。

- 国内版 App **不支持上传音频文件**，无"上传的录音"类型记录
- 记录列表为唯一存放音频的入口，**无独立的【空间】菜单**，但支持**自定义分组**（见 §1.1）

### 1.1 顶部分组视图切换

列表顶部左侧有视图切换下拉菜单（默认显示「全部记录 ▼」），点击弹出下拉列表：

| 视图选项 | 说明 |
|----------|------|
| **全部记录** | 默认视图，展示所有未删除的录音记录（含已分组和未分组的） |
| **回收站** | 展示已删除的记录（仅云同步开启时删除的记录会进入回收站） |
| **分组列表** | 展示用户自定义的各个分组（如"仅录音"、"合并测试"等），点击可查看该分组下的记录 |
| **管理分组** | 下拉列表底部的固定入口，点击进入分组管理页面 |

### 1.2 管理分组

- **入口**：顶部分组下拉列表底部 → 「管理分组」
- **功能**：支持**创建新分组**、**编辑分组名称**、**删除分组**
- **云端同步**：若开启了「云同步」，分组信息及分组内记录多端同步；若未开启「云同步」，分组仅存在于本地，且与「全部记录」在云端无区别

> 卡片显示规则、排序方式、未读红点、时间与时长格式 → 详见 `domain/record-common-rules.md`

## 二、筛选方式

| 筛选维度 | 可选项 |
|----------|--------|
| 录音类型 | 全部（默认）/ 通话录音 / 音视频录音 / 现场录音 / 实时翻译与录音 |

**筛选规则：**
- 筛选只能单选（选新项自动取消原选项）
- 通话录音筛选包含通话闪录下载的记录；现场录音筛选包含现场闪录下载的记录
- 筛选结果为空 → 显示空状态，筛选图标保持高亮，不显示录音按钮
- 选择非默认筛选项 → 筛选图标选中高亮；选回「全部」→ 恢复默认颜色
- 筛选记忆规则同排序：切换 Tab 保持；重启 App / 重登恢复默认

> 国内版无「音频位置（已加入空间/未加入空间）」筛选维度，无「上传的录音」选项。

## 三、左滑快捷操作

| 操作 | 条件 | 行为 |
|------|------|------|
| 删除 | 已上云记录 | 删除到回收站 |
| 删除 | 未上云记录 | 直接永久删除，不进回收站 |

> 国内版无「添加到空间」左滑操作。  
> 删除行为详细规则 → 详见 `domain/delete-recycle-rules.md`

## 四、批量操作

### 进入与退出

| 操作 | 说明 |
|------|------|
| 进入 | 长按任意记录卡片 |
| 退出 | 点击退出按钮 / Android 系统返回手势 |
| 解析中/解析失败记录 | **置灰不可选择，但保留卡片 UI** |
| 底部按钮 | 「合并」+「移到」+「导出」+「删除」四个按钮 |
| 未选择 | 底部按钮置灰不可点击 |

> 国内版批量操作有 4 个按钮（合并/移到/导出/删除），无「添加到空间」按钮，改为「移到」分组。

### 批量合并

- 选择多条记录后点击「合并」
- 仅支持**手动合并**（无自动合并）

#### 4.1 前置校验与限制（V6.3.0 基线）

**不支持条件与优先级（同时命中时只提示最高优先级一项）：**
1. 翻译模式录音不支持合并
2. 选择条数超过 10 条
3. 所选录音总时长超过 5 小时
4. 含未下载音频且当前断网
5. 含处理中记录

**次数限制：**
- 当天成功合并上限为 **100 次**
- 第 101 次发起时拦截并提示

#### 4.2 合并确认页规则（V6.3.0 基线）

| 项 | 规则 |
|----|------|
| 默认顺序 | 按时间从早到晚排序 |
| 顺序调整 | 支持拖拽调整拼接顺序 |
| 默认标题 | 按第一条记录生成 |
| 标题递增 | 若原标题已以「合并」或「合并+数字」结尾，按「合并2」「合并3」递增 |
| 标题编辑 | 支持手动编辑；清空后仍可合并，最终回退默认标题 |
| 「合并后保留原音频」开关 | 默认关闭，且**不记忆**上次状态 |

#### 4.3 智能标题联动（V6.3.0 基线）

- 智能标题开关默认关闭，但**记忆上次状态**
- 点亮后标题输入框置灰；再次点击开关或点击标题输入框可恢复手动编辑
- 断网点击点亮智能标题会提示 toast，但仍允许点亮
- 断网且点亮状态下点击合并，会再次 toast，并按手动/默认标题继续合并
- 联网时若智能标题生成失败，也回退到手动/默认标题

#### 4.4 合并结果规则（V6.3.0 基线）

- 合并成功后生成 1 条新的合并录音记录
- 音频和转写按用户设定顺序拼接，时间戳按合并后内容重新计算
- AI 内容（概览/摘要/待办/脑图）按合并后的完整转写重新生成
- 类型、语种、地点、分组归属按第一条记录继承
- 保留原始音频基本信息用于溯源
- 总时长应等于原录音总时长之和（允许合理误差）

#### 4.5 特殊场景（V6.3.0 基线）

- 含未下载音频但联网：可先下载再合并
- 含未转写或转写无内容录音：合并后按原状处理或直接拼接
- 说话人设置不一致：合并后保持原状
- 部分音频无 opus、部分有 opus：后续重转写/区分说话人统一使用 mp3
- 未联网且多选已上云记录，且未勾选「合并后保留原音频」：合并成功后需提示原记录删除失败

### 批量移到分组

- 选择记录后点击「移到」
- 弹出分组选择列表，可将选中的记录移动到指定分组中
- 移到分组后，记录仍在「全部记录」中可见

### 导出类型

| 操作 | 说明 |
|------|------|
| 导出音频 | 导出所选记录的音频文件 |
| 导出转写 | txt / docx / pdf（含时间戳/说话人选项）|
| 导出摘要 | txt / markdown / docx / pdf |
| 导出思维导图 | markdown / pdf |

> 导出格式、命名、进度弹窗等详细规则 → 详见 `domain/export-rules.md`

### 批量删除

选择记录后点击删除 → 已上云记录删除到回收站。

> 删除行为详细规则 → 详见 `domain/delete-recycle-rules.md`

## 五、录音入口

| 列表状态 | 入口位置 |
|---------|---------|
| 有记录 | 右下角或右上角**录音按钮** |
| 无记录（空状态） | 中部**开始录音**按钮 |

> 国内版记录列表**无上传入口**，仅支持通过录音添加新记录。

## 六、云同步相关

国内版无套餐收费的「上云」概念，但用户可在「账号设置」中开启/关闭「**云同步**」功能：

| 云同步状态 | 记录行为 |
|-----------|---------|
| **已开启** | 录音完成后自动上云；删除进入回收站；分组信息及记录多端同步 |
| **已关闭** | 录音仅保存在本地；删除为永久删除，不进回收站；分组仅在本地生效，与「全部记录」无异 |

## 七、测试关注点

- 记录列表仅展示音频类型，无上传音频记录类型
- **分组视图**：顶部下拉菜单支持切换「全部记录」「回收站」及自定义分组；可进入「管理分组」创建/编辑/删除
- **筛选**：无「音频位置」维度，无「上传的录音」选项；通话录音筛选包含通话闪录
- 筛选结果为空时显示空状态，筛选图标保持高亮
- **左滑**：只有删除操作，无「添加到空间」；已上云→回收站；未上云→永久删除
- **批量 4 按钮**：合并/移到/导出/删除（无「添加到空间」）
- **合并前置校验**：翻译模式/超 10 条/超 5 小时/未下载且断网/处理中，按优先级只提示一条
- **合并确认页**：顺序可拖拽、默认标题与递增规则、保留原音频开关默认关闭且不记忆
- **合并结果**：音频与转写按顺序拼接、时间戳重算、AI 内容重生成、属性继承第一条、总时长正确
- 批量模式下解析中/解析失败记录**置灰可见**
- 云同步关闭时，删除记录为永久删除，不进回收站；分组仅存在本地
- 无上传入口，新记录只能通过录音产生

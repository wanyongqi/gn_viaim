# Production Verification

## Quick Start

当用户要求生成或更新现网包验证清单时，按以下顺序执行：

1. 读取本次迭代的需求点或简约版测试点。
2. 读取上一版本的现网包验证清单（如存在）。
3. 对照迭代内容，识别新增/修改/删除的功能模块。
4. 更新验证清单：新增条目、调整已有条目、标注受影响老功能。
5. 输出最终验证清单，说明本版验证重点背景。

## Core Workflow

### Step 1：读取材料
- 读取本次迭代的需求点、简约版测试点或测试用例。
- 若存在上一版本验证清单（如 `630现网包验证重点.md`），一并读取作为基线。

### Step 2：识别本版变化
- 新增功能 → 新增验证条目
- 修改功能 → 更新对应条目，注明变化点
- 删除/隐藏功能 → 注明已移除，从验证清单中删除或标注
- 间接影响功能 → 在已有条目下追加"注意：xxx 受本次改动影响，需额外关注"

### Step 3：固定必验项兜底
无论本次改动内容，以下项必须出现在验证清单中：
- **录音主流程**：三类录音启停、暂停恢复、保存
- **实时转写**：转写上屏正常
- **老数据兼容**：历史录音播放/查看
- **导出功能**：单条导出（音频/转写）
- **蓝牙连接**：连接/断连/重连

### Step 4：输出验证清单
- 文档开头写"本版验证背景"（本次有哪些新功能，哪些老功能可能受影响）。
- 按功能模块分节，注明每节的验证重点。
- 每条 1-3 句话，说明验证目标，不展开用例细节。

## Output Format

参考 `630现网包验证重点.md`：

```md
# xxx 版本现网包回归验证重点

现网包验证主要关注**核心主流程的通畅性**、**新功能上线质量**以及**新老数据兼容性**。

## 1. 核心录音与转写主流程
- **录音启停与暂停**：验证三类录音的正常开启、暂停、恢复及结束保存。
- **实时转写**：转写上屏流畅，无丢字、卡顿。

## 2. [本版新增功能]
- **xxx**：验证 xxx，重点确认 xxx。

## 3. 老数据与历史兼容
- **老录音查看**：播放旧版本产生的录音，确认音频、转写、AI摘要等展示正常，无崩溃或数据丢失。

## 4. [受间接影响的老功能]
- **xxx**：本次 xxx 改动可能影响 xxx，需额外验证 xxx。
```

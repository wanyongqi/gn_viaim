## 迭代调整规则详解

### 新增功能
- 在验证清单中新增对应章节。
- 验证重点需覆盖：核心正向流程 + 至少一个边界/异常场景。

### 修改功能
- 在已有章节中更新验证重点描述。
- 明确说明本次变化点（如"新增 xxx 入口""移除 xxx 按钮"）。
- 若修改涉及存量数据，需在"老数据兼容"章节补充验证条目。

### 删除/隐藏功能
- 在验证清单中注明已移除（或仅对特定用户隐藏），无需执行。
- 若部分用户仍可见，需说明权限条件。

### 间接影响功能
- 在已有条目下追加注意事项。
- 格式：`注意：本次 xxx 改动可能影响 xxx，需额外关注 xxx。`

## 写作要点

### 验证条目写作标准
- 主语：功能/模块名称
- 谓语：验证 / 确认 / 重点确认
- 宾语：验证目标（结果可观察，不是步骤）

正确示例：
```
- **合并录音**：验证多选合并，确认合并后音频连续播放，转写与AI内容正常生成，原记录按开关正确保留或删除。
```

错误示例（过于简单，无信息量）：
```
- **合并录音**：测试合并功能。
```

错误示例（展开用例细节）：
```
- **合并录音**：
  1. 选择2-10条录音
  2. 点击合并
  3. 确认弹窗
  4. 验证结果
```

# 测试用例输出要求参考

## 输出格式

- 采用 Markdown 语法输出用例。
- 用例需包含以下字段：测试项、用例名称、优先级、前置条件、操作步骤、预期结果。

## 结构规范

- 单个需求文件默认使用列表树结构输出，不直接使用 Markdown 的 `#`、`##`、`###` 标题。
- 默认层级结构为：`- 功能点名称` -> `- 【一级分组】` -> `- 【二级分组】` -> `- tc-Px：用例标题` -> `- pc:` -> `- 步骤` -> `- 预期结果`
- 若存在分级小标题，则分级小标题本身也占用一层结构，属于该标题下的测试用例需继续下沉一级。
- 不允许把分组写成普通说明文本。

## 标题与文案

- 用例名称前缀必须为 `tc-优先级`，例如：`tc-P0：音视频录音记录不会自动生成智能标题`
- 标题需按主谓宾表达，尽量直接描述“什么场景下什么行为结果正确/异常”。
- 前置条件使用 `pc:` 前缀，浓缩为一句状态句，优先写“已进入 / 位于 / 已选中 / 已打开 / 已触发”等状态。
- 操作步骤中每个步骤对应一个预期结果，预期结果首行缩进，且无额外前缀。
- 单条用例最多不超过 3 个步骤，优先控制在 2 个步骤。
- 一条用例尽量只承载 1 个核心断言，不要把两个独立验证目标强行塞进同一条。
- `pc:` 写状态，步骤写动作，预期写可观察结果。

## 文件命名与输出位置

- 单个需求默认输出为独立 `md` 文件，文件名格式为：`需求名称_测试用例.md`
- 若当前需求明确属于某一版本，默认输出到对应版本目录，例如：`630版本/xxx_测试用例.md`
- 如需汇总多个需求到一个总文件，再额外生成汇总文件。
- 汇总文件中，每个来源需求的首行功能点名称可转为 Markdown 一级标题；单个需求文件仍保持列表树格式。

## 拆分与合并规则

- 录音类型默认分开写，除非用户明确允许合并。
- 机型、耳机型号、平台存在明确差异时，可在同一文件内分组输出；是否拆成多个文件以用户要求为准。
- Android / iOS 默认视为功能一致，除非是系统限制、交互差异或需求明确指出，否则不区分用例。
- 若需求点要求覆盖多维度，优先使用“分组下沉”而不是在单条标题中混杂多个维度。
- 用户若明确要求“3 大录音用例分开写”“不要强行合并用例”等，需优先遵守。

## 优先级定义

- `P0`：核心功能的正向测试用例。
- `P1`：基本功能的常用场景，包含重要的错误和边界值。
- `P2`：基本功能的不常用场景。
- `P3`：基本功能的反向测试用例，覆盖异常和边界。
- 优先级目标比率默认参考：`P0` 约 15%，`P1` 约 30%，`P2` 约 35%，`P3` 约 20%。
- 优先级比率默认按单个需求文件控制，允许小幅浮动；文件体量较小时，不为凑比例而强行加删用例。

## 覆盖范围

- 覆盖所有符合真实用户场景的情况，并覆盖所有功能点。
- 尽可能详尽地覆盖异常场景，确保功能稳定。
- 关注交互操作准确性及 UI 展示与设计稿一致性。
- 除主流程外，应主动考虑边界、异常、中断、恢复等场景，例如：断网、失败、切后台、最小化、kill APP、重进页面、重试、状态恢复等；如确实不适用再跳过。
- 涉及语种能力时，默认参考 `docs/source-data/语种优先级_可读版.md` 选择覆盖样本；除正向语种外，还需关注混合语种、识别失败、无内容、断网等相关场景。

## 需求分析与变更处理

- 等待用户给出需求后开始分析，不明确的地方先提问。
- 不允许新增、修改、曲解需求意思。
- 提问后等待用户回复，再次完善需求分析，并等待用户最终“开始生成”的指令。
- 若用户后续补充或修改需求，以最新口径为准；若已有用例与最新逻辑冲突，应调整或删除旧用例。

## 可选交付物

- 如用户需要，可额外输出用例总数、优先级分布、补漏建议。
- 如用户需要，也可额外生成“需求点 -> 用例标题 / 数量”的对照清单，便于评审、追溯和查漏。

## 示例

```md
- 录制过程中支持暂停录制
  - 【通话录音】
    - tc-P0：通话录音中点击暂停按钮可暂停录制
      - pc: APP已连接耳机，处于通话录音进行中
      - 点击录音界面的暂停按钮
        - 录音暂停，界面显示暂停状态，计时停止
  - 【弹窗与语言选择】
    - tc-P1：录音中打开转写语言选择弹窗交互正常
      - pc: 处于任一类型录音进行中
      - 点击转写语言入口
        - 转写语言选择弹窗正常弹出
      - 在弹窗中滚动语言列表并选择某一语言
        - 语言列表可滚动，选中后弹窗关闭
```

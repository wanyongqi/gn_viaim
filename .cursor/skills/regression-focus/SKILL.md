# Regression Focus

## Quick Start

当用户要求给出回归建议时，按以下顺序执行：

1. 读取本次迭代的需求点或测试用例文件。
2. 识别本次改动直接涉及的功能模块。
3. 识别可能被间接影响的上下游功能（公共组件、状态机、语言选择逻辑等）。
4. 对照核心主流程清单，标注每版必回归项。
5. 输出回归建议，说明每条的影响原因。

## Core Workflow

### Step 1：读取迭代内容
- 读取用户提供的需求点、简约版测试点或测试用例文件。
- 提取本次新增、修改、删除的功能点列表。

### Step 2：直接影响分析
- 逐一梳理每个改动点，找出直接涉及的功能模块和交互路径。
- 标注为：**必回归（直接影响）**

### Step 3：间接影响分析
- 判断改动是否触及以下公共路径：
  - 录音状态机（开始/暂停/继续/结束）
  - 语言选择逻辑（转写语言/翻译语言）
  - 导出功能链路
  - 列表/详情页通用交互
  - 蓝牙连接与耳机控制
- 标注为：**建议回归（间接影响）**

### Step 4：核心主流程兜底
- 每版必回归项（无论改动内容）：
  - 录音启停与暂停（三类录音）
  - 实时转写上屏
  - 录音保存与详情页查看
  - 导出功能（单条）
- 标注为：**必回归（主流程）**

### Step 5：输出建议
- 按功能模块分组，注明优先级和影响原因。
- 可选附上"本版不影响"清单。

## Output Format

```md
## 回归重点建议（xxx 版本）

### 必回归
- **功能模块**（直接影响）：原因说明
- **核心主流程**（每版必回归）：原因说明

### 建议回归
- **功能模块**（间接影响）：原因说明

### 可选回归
- **功能模块**（低优先级）：原因说明

### 本版不影响（无需回归）
- 功能模块：说明
```

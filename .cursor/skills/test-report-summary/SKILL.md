# Test Report Summary

## Quick Start

当用户要求生成测试报告功能点时，按以下顺序执行：

1. 读取本次迭代的测试用例文件或需求点列表。
2. 按功能模块分组，提炼各模块的覆盖维度。
3. 每个维度用加粗标签 + 简约描述表达，控制在 1-2 句话。
4. 输出为 Markdown 文档，文件名格式：`xxx版本简约版测试点.md`

## Core Workflow

### Step 1：读取材料
- 优先读取已完成的测试用例文件。
- 若用例未完成，可基于需求点/简约版测试点进行提炼。

### Step 2：功能模块分组
- 每个需求点对应一个功能模块（一级标题 `##`）。
- 编号按需求点顺序排列（如 `## 1. 录制过程中可暂停录制`）。

### Step 3：覆盖要点提炼
- 按以下维度扫描用例，归纳出覆盖要点：
  - **基础控制**：核心正向流程覆盖了什么
  - **语言/设置切换**：语言、配置类功能覆盖了什么
  - **异常与边界**：异常场景、边界值覆盖了什么
  - **状态控制**：状态管理、并发、隐藏/禁止类功能
  - **兼容性与UI**：多模式、多平台、样式适配覆盖了什么
  - **多端同步**：跨端一致性（如适用）
- 不是每个维度都必须出现，按实际覆盖情况选取。

### Step 4：输出文档
- 使用 Markdown 标题 `##` 分隔各功能模块。
- 二级要点使用 `- **标签**：描述` 格式。
- 每条控制在 1-2 句话，不展开用例细节。

## Output Format

参考 `630简约版测试点.md`：

```md
## 1. 功能模块名称

- **基础控制**：覆盖通话、现场、音视频三类录音的 xxx 操作，以及 xxx 状态的正确恢复。
- **语言切换**：xxx 期间切换转写/翻译语言，xxx 后新语言即时生效。
- **异常与边界**：覆盖最小化返回、杀进程结束并保存、锁屏恢复、xxx 等场景。
- **状态控制**：隐藏 xxx、禁止 xxx、xxx 自动保存、xxx 横幅提示及恢复。
- **兼容性与UI**：浅色/深色/英文模式适配，以及 xxx 的同步更新。
```

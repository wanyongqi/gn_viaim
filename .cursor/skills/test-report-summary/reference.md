# 测试报告功能点梳理参考

## 格式规范

### 文档结构
```md
# xxx版本 简约版测试点

## 1. 功能模块名称（与需求点对应）

- **维度标签**：简约描述（1-2句）

## 2. 功能模块名称

- **维度标签**：简约描述
```

### 维度标签参考
根据实际覆盖情况选用，不强求所有维度都出现：

| 标签 | 适用场景 |
|------|----------|
| **基础控制** | 核心正向流程、操作启停 |
| **语言切换** | 转写语言、翻译语言、语种记忆 |
| **入口与默认值** | 功能入口、默认状态、初始展示 |
| **识别能力** | 转写准确性、语种识别、混合语种 |
| **异常与边界** | 异常场景、边界值、错误处理 |
| **状态控制** | 并发控制、隐藏/禁止入口、状态保持 |
| **进度与异常** | 进度展示、失败处理、中断恢复 |
| **多端同步** | APP/Web/桌面端一致性 |
| **兼容性与UI** | 深色/浅色/英文模式、布局适配 |
| **权限控制** | 用户权限隔离、白名单/非白名单 |
| **交互与跳转** | 点击跳转、弹窗交互、页面切换 |
| **前置校验** | 合法性校验、上限控制、禁止条件 |

## 写作要点

### 正确示例
```md
- **基础控制**：覆盖通话、现场、音视频三类录音的暂停、继续操作，以及仅录音与实时转写状态的正确恢复。
- **异常与边界**：覆盖最小化返回、杀进程结束并保存、锁屏恢复、多次暂停/继续、快速连点等场景。
```

### 错误示例（过于具体，照搬用例）
```md
- tc-P0：通话录音点击暂停可暂停录制
  - pc: 耳机已连接，通话录音进行中
  - 点击暂停
    - 录音暂停，界面显示暂停状态，计时停止
```

### 错误示例（过于模糊，无信息量）
```md
- **基础控制**：测试了录音相关功能。
```

---

## 信息密度参考标准

- 每个功能模块的覆盖要点：3-8 条
- 每条描述：1-2 句话
- 每句话：15-40 字
- 不出现具体操作步骤、预期结果、用例编号

---

## 文件命名与输出位置

- 文件名格式：`xxx版本简约版测试点.md`（如 `630简约版测试点.md`）
- 输出位置：对应版本目录（如 `630版本/`）
